static-render

Pre-render web components at build time. Zero changes to your components.

Every demo below was pre-rendered at build time using static-render. Disable JavaScript to see the static content, or leave it enabled to watch the components hydrate with fresh data.

Demos

  • Shadow DOM — components using attachShadow, fetching from an API
  • Light DOM — a replies component that appends to light DOM
  • RSS Feed — remote component script loading a Mastodon RSS feed

How it works

  1. At build time, your HTML + component JS runs in a server-side DOM
  2. Components make real API calls and render their content
  3. Shadow DOM is serialized as Declarative Shadow DOM
  4. A tiny hydration shim ensures a smooth swap when JS loads in the browser
